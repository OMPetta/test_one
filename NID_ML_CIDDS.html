Flags¶

The 'flags' field is categorical and lists Transmition Control Protocol flags. We have 28 categories. Each character in the string represents the first letter of the flag type identified in the traffic- (APRSF):Acknowledgement (ACK) here as "A", Push (PSH) here as "P", Reset (RST) here as "R", Synchronization (SYN) here as "S", Finish (FIN) here as "F". There are several hexadecimal representations of flags occuring with less frequency. These data can be traslated using a TCP flag key such as this here: http://rapid.web.unc.edu/resources/tcp-flag-key/ . These data with hexadecimal representations contain the common TCP flags(APRSF), as well as the less common ECN-Echo (ECE) and Congestion Window Reduced (CWR) flags. For the purpose of this study, traffic with these flags will be removed.¶

Label¶

All traffic to the ports 80 and 443 could be normal traffic or intrusion attempts. As a consequence, traffic to the ports 80 and 443 on the external server is labelled as unknown. These data will not be neccasry and are problematic to our purposes. They will need to be removed. Additionally, the study is intended to differentiate between normal and attack traffic. As such, traffic not labeled in these categories will need to be removed.¶
